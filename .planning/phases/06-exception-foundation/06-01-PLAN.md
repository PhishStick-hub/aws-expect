    ```python
    class StopConditionError(Exception):
        """Raised when a ``stop_when`` predicate itself raises an exception.

        Wraps the original exception via ``__cause__`` (standard Python
        exception chaining).  Use ``raise StopConditionError(id) from exc``.

        Does NOT inherit :class:`WaitTimeoutError` — a predicate crash is
        a logic error, not a timeout condition.

        Attributes:
            resource_id: Identifier of the resource being polled when the
                predicate crashed (e.g. ``"bucket/key"``, ``"table_name"``).
        """

        def __init__(self, resource_id: str) -> None:
            self.resource_id = resource_id
            cause_text = str(self.__cause__) if self.__cause__ else "unknown error"
            super().__init__(
                f"Error in stop_when predicate for {resource_id}: {cause_text}"
            )


    class StopConditionMetError(Exception):
        """Raised when a ``stop_when`` predicate returns ``True``.

        Signifies that the target resource entered a terminal state from
        which it will never satisfy the success condition.  Polling stops
        immediately — no further sleep or retry.

        Does NOT inherit :class:`WaitTimeoutError` — the condition was
        *met*, not *timed out*.  Code that catches ``WaitTimeoutError``
        will *not* catch ``StopConditionMetError``.

        Attributes:
            resource_id: Identifier of the resource whose stop condition
                was triggered (e.g. ``"bucket/key"``, ``"table_name"``).
            stop_reason: Descriptive string explaining why polling stopped.
                This is the string-typed return value of the predicate, or
                a generic reason when the predicate returned ``True``.
            elapsed: Number of seconds that had elapsed when the stop
                condition fired.
            timeout: The timeout originally configured for the wait
                operation (in seconds).
        """

        def __init__(
            self,
            resource_id: str,
            stop_reason: str,
            elapsed: float,
            timeout: float,
        ) -> None:
            self.resource_id = resource_id
            self.stop_reason = stop_reason
            self.elapsed = elapsed
            self.timeout = timeout
            super().__init__(
                f"Stop condition met for {resource_id} after {elapsed}s"
                f" (timeout={timeout}s)\nStop reason: {stop_reason}"
            )
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User predicate → StopConditionError | User-provided callable raises arbitrary exception; wrapped via raise-from. No trust assumption — predicate is untrusted code. |
| User predicate return → StopConditionMetError | User-provided callable returns `bool | str`; str value becomes `stop_reason` in exception message. Reason string is treated as opaque — no parsing, no sanitization beyond Python's default exception rendering. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | StopConditionError.__str__ | mitigate | The predicate exception text is included in __str__ via `str(__cause__)`. Python's default `str()` on exceptions already limits output length and does not include stack traces. No additional sanitization needed — the predicate author controls what their exception reveals, and they're the caller of the library. |
| T-06-02 | Denial of Service | StopConditionMetError.__init__ | accept | User-controlled `stop_reason` string could theoretically be gigabytes. However, this is the caller's own data flowing back to them — self-DoS. No PII or sensitive data processed. Python string allocation limits provide natural bound. |
| T-06-03 | Tampering | StopConditionMetError attributes | accept | All attributes are set in __init__ via positional args. No setters, no mutable defaults, no deserialization path. After construction, attributes are effectively immutable (convention, not enforcement). Risk profile is identical to all existing exception classes in the codebase. |
| T-06-04 | Elevation of Privilege | Exception hierarchy | mitigate | `StopConditionMetError` deliberately inherits `Exception` (not `WaitTimeoutError`) so that `except WaitTimeoutError` handlers do NOT catch it. This is the intended security property — stop-condition failures must NOT be silently swallowed by generic timeout handlers. Verified by test assertion. |
</threat_model>
